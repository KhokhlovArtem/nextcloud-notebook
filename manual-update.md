# Ручное обновление Nextcloud (пошаговая инструкция)

**Цель:** Безопасно обновить Nextcloud до актуальной версии с сохранением всех данных и настроек.

**Время выполнения (в среднем):** 20-40 минут (без учёта создания резервной копии).

---

## ⚠️ Перед началом работы

- Убедитесь, что у вас есть **доступ по SSH** к серверу.
- Убедитесь, что на сервере достаточно места на диске (минимум в 2 раза больше размера папки `nextcloud`).
- Прочтите [официальные заметки к выпуску](https://nextcloud.com/changelog/) — иногда требуются действия с БД или PHP.

---

## Шаг 1. Снимаем внеплановую резервную копию

Перед любым обновлением **обязательно** создайте полную резервную копию.

Создаем средствами СРК или средствами приложения

```bash
# Переменные (измените под себя)
NEXTCLOUD_PATH="/var/www/nextcloud"
BACKUP_PATH="/backups/nextcloud_pre_update_$(date +%Y%m%d_%H%M%S)"

# Создаём папку для бекапа
mkdir -p $BACKUP_PATH

# 1.1 Копируем файлы
cp -r $NEXTCLOUD_PATH $BACKUP_PATH/files

# 1.2 Экспортируем БД (MySQL/MariaDB)
mysqldump -u root -p nextcloud_db > $BACKUP_PATH/database.sql

# 1.3 Сохраняем конфиги (на всякий случай)
cp $NEXTCLOUD_PATH/config/config.php $BACKUP_PATH/
```

**Проверка:** Убедитесь, что файл `database.sql` не пустой (весит хотя бы несколько килобайт).

---

## Шаг 2. Переводим сервер nextcloud в режим обслуживания.

Ставьте сервер в режим обслуживания **до начала**:

### 2.1 Перейдите в папку Nextcloud

```bash
cd /srv/nextcloud/    # укажите ваш путь
```

### Шаг 2.2 Переводим сервер nextcloud в режим обслуживания.

```
sudo -u www-data php occ maintenance:mode --on
```

## Шаг 3. Обновление операционной системы

Переключитесь на `root` или используйте `sudo`.

```bash
# Для Debian/Ubuntu
sudo apt update && sudo apt upgrade -y

# Для CentOS/Rocky/Alma
sudo dnf update -y
```

**Обязательно:** После обновления ОС перезагрузите сервер, если обновилось ядро или важные библиотеки.

```bash
sudo reboot
```

После перезагрузки проверьте версии PHP, MySQL и веб-сервера:

```bash
php -v
mysql --version
nginx -v   # или apache2 -v
```

---

## Шаг 4. Обновление Nextcloud

X### 4.1 Переключитесь на пользователя веб-сервера

```bash
sudo -u www-data bash   # Debian/Ubuntu
# или
sudo -u apache bash     # CentOS/RHEL
```

### 4.2 Перейдите в папку Nextcloud

```bash
cd /srv/nextcloud/   # укажите ваш путь
```

### 4.3 Запустите ручное обновление через updater

```bash
sudo -u www-data php updater/updater.phar --no-backup
```

> **Почему `--no-backup`?**  
> Мы уже сделали резервную копию вручную (Шаг 1).

### 4.4 Что вы увидите в консоли

- Запустится проверка окружения.
- Апдейтер предложит скачать новую версию.
- После скачивания — спросит разрешение на применение изменений (нажмите `y`).
- Будет выполнена миграция базы данных.
- В конце появится сообщение об успехе.

### 4.5 Если возникла ошибка

**Типичная проблема:** Недостаточно прав на запись.

```bash
# Выйдите из sudo -u (Ctrl+D) и выполните от root:
chown -R www-data:www-data /var/www/nextcloud
chmod -R 755 /var/www/nextcloud
```

После исправления повторите шаг 4.3.

## Шаг 5. Завершающие действия

### 5.1 Очистите кеш

```bash
sudo -u www-data php occ maintenance:repair
sudo -u www-data php occ db:add-missing-indices
```

### 5.2 Проверьте версию

Зайдите в веб-интерфейс Nextcloud → Настройки админа → внизу страницы будет версия.

Или через консоль:

```bash
sudo -u www-data php occ status
```

### 5.3 Перезапустите веб-сервер и PHP-FPM (если используется)

```bash
sudo systemctl restart nginx php8.3-fpm.service   # пример для Ubuntu
```

## Шаг 6. Переводим сервер nextcloud в режим работы .

```bash
sudo -u www-data php occ maintenance:mode --off
```

- [ ] Веб-интерфейс Nextcloud открывается без ошибок.
- [ ] Все пользователи могут залогиниться.

  ```

## 🚨 Откат на случай провала

Если после обновления всё сломалось:

```bash
# Останавливаем веб-сервер
sudo systemctl stop nginx

# Восстанавливаем файлы из бекапа
rm -rf /var/www/nextcloud
cp -r $BACKUP_PATH/files /var/www/nextcloud

# Восстанавливаем БД
mysql -u root -p nextcloud_db < $BACKUP_PATH/database.sql

# Запускаем сервер обратно
sudo systemctl start nginx
```

После отката — разбирайтесь с логами и не пытайтесь обновиться снова без диагностики.

---

## 📌 Важные замечания

- **Никогда** не обновляйтесь через `apt upgrade` для Nextcloud (если ставили не из официального репозитория — всё поломается).
- **Обновляйте только через мажорные версии постепенно** (например, 25 → 26 → 27). Прыжок с 25 на 28 часто убивает базу данных.
